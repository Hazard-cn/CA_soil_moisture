# 两次网页端审阅的结构化综合与校正

## 来源性质

本文件综合两次由用户发起的网页端审阅：`6a5728e2-7ccc-83ec-a1ff-afded5df2758`（文献方法导向优化）和`6a572891-0c3c-83ec-b36e-c8efa69ce72f`（独立审阅与方案优化）。原始回复属于外部建议，不是项目机器事实；本包只保留经本地报告、代码或机器表核对后的问题与方案边界，不保存原始对话文本。

## 已确认应进入下一轮的意见

1. G185必须报告低、高SR总条件斜率，不能仅凭P75−P25为正就统一称为损失缓冲。
2. 事件稿必须区分季节累计事件天数与单次事件持续期，并同时报告低SR、高SR和二者差值的联合区间。
3. 事件产量主设计没有事件指示变量；若要解释严重程度边际，需要新canonical ID和两部分模型。
4. 90%SM恢复端点在`t=0`高度集中，不应继续调阈值救援。
5. 当前阈值方向应保持`REVIEWED_NOT_CANDIDATE`；未来可迁移性或预测验证属于新研究问题。
6. 外生阈值栅格与本项目逐日Tmax °C-day实现不等同于公开逐小时算法。

## 对第一份brainstorm的关键校正

第一份审阅把“G185固定样本、事件稿固定热干事件、阈值稿外生阈值”同时放入S1冻结包。该拼接不可直接执行：G185和V3不是同一数据家族；固定事件与空间阈值暴露也不是同一定义。可进入新设计审查的S1概念应限定为单一V3家族、固定32°C热干事件、GLEAM SMrz状态和统一SR端点，不得加入G185样本或外生连续阈值。但该概念尚不是可直接执行的冻结包：必须先从[数据家族与estimand边界](../methods/data_family_estimand_matrix.md)列出的三个互斥SM时序方案中选择一个，并保证无事件年份的状态可定义、多个事件不导致同一grid-year产量被复制。完成该冻结前，S1只能作为follow-up建议。

## 对第二份整合稿建议的关键校正

“暴露表征如何改变推断”是可能的新研究问题，但不能直接把现有三套结果排列后综合。只有在同一V3共同样本、同一物候窗口、同一SR端点、同一FE和空间推断下重新构造季节指标、事件指标和外生阈值指标，才能比较暴露表征。该研究需要新canonical ID和结果盲态设计审查。

## 建议优先级

事件两部分模型的重新冻结优先于新增独立稿；先行SM状态只能作为待完成时序和多事件聚合冻结的新canonical次级模块。G185在端点分类和强FE披露后可保留为独立备选或季节指标基准。阈值当前版本只作为失败证据；未来若新增年份、逐小时数据或独立验证样本，再考虑外生阈值可迁移性研究。
