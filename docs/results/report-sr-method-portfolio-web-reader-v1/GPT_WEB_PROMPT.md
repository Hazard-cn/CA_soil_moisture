# 可直接提交给网页端GPT的任务说明

请先读取本目录的[统一入口](report.md)，再按其中的“核心阅读顺序”阅读状态表、外部审阅更新、方法边界、三个既有方向的公开报告和37篇文献索引。其余Markdown和自包含HTML只用于核对具体数字、审查状态或图表，不要求逐字综合。请先把材料分为“机器或公共报告事实”“外部审阅提出但尚未执行的问题”“可供brainstorm的新方案”，不得把三类材料混写。

请对三个现有方向进行严格独立审阅，并重点处理以下问题：G185低、高SR端点斜率与province×year结果是否支持当前标题；事件稿没有事件指示变量时，现有P50→P90 estimand应如何准确命名；如何建立事件发生、事件阳性累计天数和事件阳性强度的两部分模型；90%恢复端点退化后应使用哪些不依赖阈值的SM动态指标；先行SM如何对无事件年份定义、如何处理一季多事件且不复制grid-year产量；当前阈值方向失败后，何种新研究问题才构成真正的新canonical设计。

随后基于37篇文献提出最多5个方案，优先考虑能够完全在单一V3数据家族中执行、与现有连续响应面不同、必须报告五个产区、包含先行或动态土壤水分证据并允许预设物候窗口的方案。不得按显著性选择阈值、窗口、区域、样本、FE或推断方式；不得把G185、固定热干事件和外生阈值直接拼接。

每个方案必须固定输出以下字段：

1. 唯一研究问题和唯一主estimand；
2. 数据家族、样本谓词和预期支持门槛；
3. 暴露定义、单位、端点和SR端点；
4. SM时序、物候窗口和五区处理；
5. 模型、固定效应、空间推断和多重检验family；
6. 三张主图和三张主表；
7. STOP规则与失败时的公开方式；
8. 与现有三个方向的重叠度和不可替代新增信息；
9. 引用的37篇文献rank、代码状态以及实际可移植的操作；
10. 数据可执行性、主要风险、预计两周内可完成的最小版本和投稿定位。

请使用“损失缓冲、跨零反转、正向条件斜率增强、负向恶化、不精确”的统一分类，并分别报告“点估计形态”和“是否在同一主推断下得到统计确认”；不得仅凭高减低差值区间不跨0确认损失缓冲、跨零反转或正向增强，不把条件关联写成因果效果。评分不奖励显著性。最后给出明确优先级：一个主推进方向、一个次级模块、一个备选稿和应停止的方向，并列出进入下一轮执行前必须修复的Critical和Major。

阈值方向的旧“每区至少80%有效阈值覆盖”机械门槛已被研究者明确取消，网页端不得把它作为现有冻结STOP规则自动恢复。可以依据支持域和外推风险提出新的诊断或门槛，但必须标记为待研究者批准的新设计建议，不能追溯性改变当前失败报告。
