# 热干事件方向：对齐逐日源只读审计

本审计发生在事件计算和产量模型之前，只检查文件可读性、日历与坐标。

| 年份 | 日数 | 温度文件 | 对齐降水文件 | 对齐GLEAM文件 |
|---|---:|---:|---:|---:|
| 2016 | 366 | 204,661,068 bytes | 50,423,881 bytes | 421,212,024 bytes |
| 2017 | 365 | 201,442,593 bytes | 48,336,004 bytes | 426,789,100 bytes |
| 2018 | 365 | 203,603,563 bytes | 48,698,484 bytes | 411,146,203 bytes |
| 2019 | 365 | 202,124,071 bytes | 48,681,861 bytes | 407,615,508 bytes |

每年三个分析文件均为376×616空间格网，纬度从53.53递减至16.03，经度从73.58递增至135.08；对应年份的坐标数组逐值差为0。2016三个日历均为1月1日至12月31日366天，2017—2019各为1月1日至12月31日365天，日期数组逐值一致。温度 `t2m_max` 为°C，对齐降水 `pre` 为mm/day，对齐GLEAM `SMrz` 为m3/m3。

V3为69,038行、22,180个grids，`grid_id year`重复数为0。把V3 float32坐标还原到唯一温度参考中心时，最大纬度差为 `1.8310550e-6` 度，最大经度差为 `7.3242160e-6` 度；因此设计采用 `1e-5` 度容差。原始CHM-PRE和全球GLEAM中心与项目参考中心约差0.02°/0.03°，不能直接用于声称精确格网连接；主分析冻结使用已经对齐温度格网的逐年产品。

本审计尚未计算事件、SM下降、恢复或产量回归。正式Stage 1前仍须计算12个分析文件的MD5/SHA-256，并按每年每区20个grid-days抽查原始索引和值。
